# French Diving Competition PDF Structure Analysis

## Overview

This document analyzes the structure of French diving competition result PDFs, specifically the **FFN (Fédération Française de Natation)** format as generated by **DiveRecorder** software.

**Source PDF**: `20251123 Championnats IDF hiver 3m-HV - Résultats détaillés.pdf`

---

## 1. PDF Header / Competition Metadata

### Competition Title Block (Page 1, Top)
```
Championnats Île-de-France 3m et HV Hiver
2025

Stade Maurice Thorez

Montreuil
dimanche 23 novembre 2025
```

**Pattern Components:**
- **Competition Name**: "Championnats Île-de-France 3m et HV Hiver"
- **Year**: "2025" (separate line)
- **Venue**: "Stade Maurice Thorez"
- **City**: "Montreuil"
- **Date**: "dimanche 23 novembre 2025" (French format: weekday + DD + month name + YYYY)

### Regex Patterns for Metadata:
```python
# Competition name pattern (French championships)
COMPETITION_NAME_PATTERN = r'(?:Championnat|Championnats)\s+(?:[\w\-]+\s+)*(?:3m|1m|HV|10m|Tremplin|Haut-Vol)'

# French date pattern
FRENCH_DATE_PATTERN = r'(?:lundi|mardi|mercredi|jeudi|vendredi|samedi|dimanche)\s+(\d{1,2})\s+(janvier|février|mars|avril|mai|juin|juillet|août|septembre|octobre|novembre|décembre)\s+(\d{4})'

# Venue patterns
VENUE_PATTERNS = [
    r'(?:Stade|Piscine|Centre|Complexe)\s+[A-ZÀ-ÿ][a-zà-ÿ]+(?:\s+[A-ZÀ-ÿ][a-zà-ÿ]+)*',
]
```

## 2. Event Headers

### Event Header Format
Events are separated by category headers:

```
Jeunes - Garçons Minimes B - 3m
Elite - Dames - 3m
Elite - Messieurs - 3m
Elite - Dames - HV
Elite - Messieurs - HV
```

**Structure**: `{Category} - {Gender/Age Group} - {Height/Event Type}`

### Category Types:
| French Term | English Translation |
|-------------|---------------------|
| Jeunes | Youth |
| Elite | Elite (Senior) |
| Junior | Junior |
| Senior | Senior |
| Master | Masters |

### Gender/Age Groups:
| French Term | Description |
|-------------|-------------|
| Dames | Women |
| Messieurs | Men |
| Garçons | Boys |
| Filles | Girls |
| Minimes A/B | Age category (12-14) |
| Cadets | Age category (15-16) |
| Juniors | Age category (17-18) |

### Height/Event Types:
| Code | Meaning | Platform Heights |
|------|---------|------------------|
| 1m | 1 meter springboard | - |
| 3m | 3 meter springboard | - |
| HV | Haut Vol (High Diving) | 5m, 7.5m, 10m |
| 10m | 10 meter platform | - |

### Regex Pattern for Event Headers:
```python
EVENT_HEADER_PATTERN = re.compile(
    r'^((?:Elite|Jeunes|Junior|Senior|Master|Catégorie|Open)[\s\-]+(?:[A-Za-zÀ-ÿ\s\-]+)[\s\-]+(?:3m|1m|HV|10m|Tremplin|Haut-Vol|Synchro))',
    re.IGNORECASE | re.UNICODE
)
```

## 3. Column Header Row

### Standard Column Structure
```
N° HauteurCoe J1 J2 J3 J4 J5 J6 J7 Cumul Points Points Pen
```

**Note**: OCR often merges "Hauteur" and "Coe" (Coefficient/DD) into "HauteurCoe"

### Column Definitions:
| Column | French | English | Description |
|--------|--------|---------|-------------|
| N° | Numéro | Number | Dive number (1-6, typically) |
| Hauteur | Height | Height | Platform height (1, 3, 5, 7.5, 10) |
| Coe/Coef | Coefficient | DD | Degree of Difficulty (1.0-4.5) |
| J1-J7 | Juge 1-7 | Judge 1-7 | Individual judge scores (0.0-10.0) |
| Cumul | Cumul | Cumulative | Sum of counting judge scores |
| Points | Points | Points | Dive score (Cumul × DD) |
| Points | Points | Total | Running total |
| Pen | Pénalité | Penalty | Penalty code (1-4) |

## 4. Athlete Result Block

### Athlete Header Line Format
```
{Rank} {Firstname} {LASTNAME} ({BirthYear}) -- {Club Name}
```

**Examples from PDF:**
```
1 Camille ROUFFIAC (2011) -- Kingfisher Club Plongeon Montr
2 Léonardo NOTO (2011) -- Kingfisher Club Plongeon Montr
1 Fanny BOUVET (1994) -- Kingfisher Club Plongeon Montr
1 Raphaël VIET TRIEM TONG (1991) -- Kingfisher Club Plongeon Montr
4 Manuel POTOR (1989) -- Paris Aquatique
```

### Athlete Line Patterns:

#### Pattern 1: DiveRecorder Format (Primary)
```python
# Format: "1 Firstname LASTNAME (2005) -- Club Name"
ATHLETE_DIVERECORDER_PATTERN = re.compile(
    r'^\s*(\d{1,2})\s+([A-ZÀ-ÿ][a-zà-ÿ]+(?:[-\s][A-ZÀ-ÿ][a-zà-ÿ]+)*)\s+([A-ZÀ-Ÿ]{2,}(?:[-\s][A-ZÀ-Ÿ]+)*)\s*\((\d{4})\)\s*[-–]+\s*(.+?)\s*$',
    re.UNICODE
)
# Groups: (1)=rank, (2)=firstname, (3)=LASTNAME, (4)=year, (5)=club
```

#### Pattern 2: FFN Format (Alternative)
```python
# Format: "1. LASTNAME Firstname (2005) Club Name 245.30"
ATHLETE_FFN_PATTERN = re.compile(
    r'^\s*(\d{1,2})[\.\s]+([A-ZÀ-Ÿ]{2,}(?:[-\s][A-ZÀ-Ÿ]+)*)\s+([A-ZÀ-ÿ][a-zà-ÿ]+(?:[-\s][A-ZÀ-ÿ][a-zà-ÿ]+)*)\s*(?:\((\d{4})\))?\s*(.+?)?\s*(\d+[\.,]\d+)?\s*$',
    re.UNICODE
)
# Groups: (1)=rank, (2)=LASTNAME, (3)=firstname, (4)=year, (5)=club, (6)=total_score
```

### Club Name Examples:
- "Kingfisher Club Plongeon Montr" (truncated "Montreuil")

- "H2024 Plongeon & Natation CAO"

## 5. Dive Result Rows

### Dive Row Format
```
{DiveCode} {Description} {Height} {DD} {J1} {J2} {J3} {J4} {J5} {Cumul} {DivePoints} {TotalPoints} {Penalty?}
```

**Examples from PDF:**
```
101C_ plongeon ordinaire avant 3 14 6,0 45 50 50 5,0 15,0 21,00 21,00
401C plongeon retoumé 3 13 35 25 30 30 3,0 90 1170 32,70
5231D saut périlleux et demi arrière avec % vrille 3 20 75 65 75 70 6,5 210 42,00 199,60
5132D saut périlleux et demi avant avec 1 vrille 3 21 0,0 00 00 00 0,0 00 0,00 148,20 1
```

### Dive Code Structure

#### Format: `{Group}{Somersaults}{Twists}{Position}`

| Component | Values | Description |
|-----------|--------|-------------|
| Group | 1-6 | Dive group number |
| Somersaults | 0-4 | Number of somersaults (×½) |
| Twists | 0-4 | Number of twists (×½) for group 5 |
| Position | A, B, C, D | Body position |

### Dive Groups:
| Code | Group Name (English) | Group Name (French) |
|------|---------------------|---------------------|
| 1XX | Forward | Avant |
| 2XX | Back | Arrière |
| 3XX | Reverse | Renversé |
| 4XX | Inward | Retourné |
| 5XXX | Twist | Vrille (Tire-bouchon) |
| 6XX | Armstand | Équilibre |

### Body Positions:
| Code | Position (English) | Position (French) |
|------|-------------------|-------------------|
| A | Straight | Tendu |
| B | Pike | Carpé |
| C | Tuck | Groupé |
| D | Free | Libre |

## 6. Numerical Values

### Difficulty (DD/Coefficient) Range:
- **Minimum**: 1.0 (basic dives)
- **Maximum**: 4.5 (complex platform dives)
- **Precision**: 0.1 increments
- **OCR Note**: Comma used as decimal separator in French (1,5 = 1.5)

### Judge Score Range:
- **Range**: 0.0 to 10.0
- **Precision**: 0.5 increments
- **Valid Values**: 0, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0, 4.5, 5.0, 5.5, 6.0, 6.5, 7.0, 7.5, 8.0, 8.5, 9.0, 9.5, 10.0

### Panel Sizes:
- **5 judges**: Standard regional competitions
- **7 judges**: National/international competitions

### Height Values (for HV events):
- 5 (5m platform)
- 7.5 or 75 (7.5m platform) - OCR may read as "75" 
- 10 (10m platform)

### Score Calculation:
```
Cumul = Sum of middle 3 scores (5 judges) or middle 5 scores (7 judges)
Dive Points = Cumul × DD
Total Points = Running cumulative total
```

## 7. Penalty Codes

### Footer Legend:
```
Codes des pénalités : 
1 = Plongeon manqué (Failed dive)
2 = Faux départ, -2 points (False start, -2 points)
3 = Position d'exécution, 2 points max
4 = Position des bras, 4/4 points max
```

### Penalty Indicators in Data:
- Penalty code appears at end of dive row
- Example: `5132D saut périlleux... 0,00 148,20 1` (penalty code "1" = failed dive)
- Failed dives show all zeros: `0,0 00 00 00 0,0 00 0,00`

## 11. Key Observations for Parser

1. **Name Order**: DiveRecorder uses `Firstname LASTNAME` format, not `LASTNAME Firstname`
2. **Separator**: Double dash `--` separates athlete name from club
3. **Birth Year**: Always in parentheses `(YYYY)`
4. **Decimal Separator**: French uses comma `,` not period `.`
5. **OCR Issues**: 
   - "A" in dive codes often OCR'd as "4"
   - Trailing characters like `_` appear after dive codes
   - "retoumé" instead of "retourné" (OCR error)
   - Numbers may merge (e.g., "1170" should be "11,70" or "11.70")
6. **Height in HV Events**: Height column varies (5, 7.5, 10) unlike fixed 3m for springboard
7. **Judge Count**: This PDF uses 5 judges (J1-J5), but column header mentions J1-J7

## 12. Summary of Regex Patterns for worker.py

```python
# Event header
EVENT_PATTERN = re.compile(
    r'^((?:Elite|Jeunes|Junior|Senior|Master)[\s\-]+(?:Dames|Messieurs|Garçons|Filles)[\s\-]+(?:[A-Za-zÀ-ÿ\s]+[\s\-]+)?(?:3m|1m|HV|10m))',
    re.IGNORECASE | re.UNICODE
)

# Athlete line (DiveRecorder format)
ATHLETE_PATTERN = re.compile(
    r'^\s*(\d{1,2})\s+([A-ZÀ-ÿ][a-zà-ÿ]+(?:[-\s][A-ZÀ-ÿ][a-zà-ÿ]+)*)\s+([A-ZÀ-Ÿ]{2,}(?:[-\s][A-ZÀ-Ÿ]+)*)\s*\((\d{4})\)\s*[-–]+\s*(.+)',
    re.UNICODE
)

# Dive code
DIVE_CODE_PATTERN = r'\b([1-6]\d{2,3}[A-Da-d])\b'

# Height extraction from event name
HEIGHT_PATTERN = r'\b(1m|3m|5m|7\.5m|10m|HV)\b'

# Judge score validation
def is_valid_judge_score(n: float) -> bool:
    return 0 <= n <= 10 and (n * 2) == int(n * 2)

# DD validation
def is_valid_dd(n: float) -> bool:
    return 1.0 <= n <= 4.5
```
